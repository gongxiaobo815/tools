Compiling an Operator

Compiling an Operator

After computation, the operator will be sent to TE. The auto_schedule interface of the TE is used to generate a schedule object. The schedule can be seen as how to efficiently execute the described computation process on hardware. That is, the related computations are mapped to the corresponding instructions on a hardware device.

The schedule object contains an IR, which uses codes similar to pseudo codes to describe the computation process. You can print the object by using related parameters. For details about the schedule, see the TVM Documentation.

TE builds the operator based on the schedule and generates binary files to be executed on hardware.

The code is as follows:

# Define the schedule.
with tvm.target.cce():
    sch = topi.generic.auto_schedule(res)

# Define build parameters.
config = {"print_ir" : True,
    "need_build" : True,
    "name" : "cce_sign",
    "tensor_list": [data, res]
}

# Build the operator and generate target files.
te.lang.cce.cce_build_code(sch, config)

NOTE:

  • with tvm.target.cce（） indicates that the specified hardware target is the CCE platform.
  • config refers to a map for setting parameters related to the operator building.
    • print_ir indicates whether to print the IR, . The default value is True.
    • need_build indicates whether to build the operator. The default value is True.
    • name indicates the operator name, which will be used in the subsequent operator calling. The default value is cce_op.
    • tensor_list refers to the list of input and output tensors of the operator. The input is the tensor object returned by the placeholder interface, and the output is the computed tensor object.
