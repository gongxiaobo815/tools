Manual Installation

Manual Installation

In this mode, you need to manually install Mind Studio and the DDK respectively.

  1. Decompress the installation package.

    Decompress the mini_mind_studio_*.rar package to obtain the Mind Studio installation package Mind-Studio_Ubuntu-x86-64.tar.

    Upload the file Mind-Studio_Ubuntu-x86-64.tar to the home directory on Linux of the common user where Mind Studio is to be installed.

    Run the tar -xvf Mind-Studio_Ubuntu-x86-64.tar command to decompress the package.

  2. Configure the configuration file.

    Go to the scripts directory that is generated after decompression and find the parameter configuration file env.conf.

    The configuration file contains the following parameters: ip, port, toolpath, backup, profiler_port, install_user, use_eth0, and package_path. They are described as follows:

    • ip: IP address used for starting Mind Studio

      NOTE:
      This parameter must be set during installation.
      If ip is set to any, the IP address of the network adapter is automatically obtained during startup.

    • port: listening port of Mind Studio

    • toolpath: path of required tools during the execution, for example, database path db.
    • backup: path of the backup files of project, my-model, and my-datasets.

    • profiler_port: listening port of Profiling

      NOTE:

      • Ensure that the port is not in use. You can run the netstat -anp | grep portnumber command to view the port number. Some ports cannot be customized in the Chrome browser （for example, ports 6666, 6000, and 6667）. You are advised to set this parameter to a value greater than 7000.
      • Ensure that these ports are not occupied: port 4412 for exec, 4411 for terminal, 4403 for wsagent_debug, 4401 for wsagent, 3002 for hiai-agent, 3001 for cce_profiler, 3003 for hiai_log, and profiler_port.

    • max_log_size: maximum number of log files that can be set by users in the ~/tools/log/hiai_agent_log folder. The default value is 10. Considering the impact of disk space, you are not advised to set this parameter to a large value.

    • apache_user: user name of the Apache service on the Profiling server

    • install_user: current installation user

      NOTE:
      This parameter must be set during installation.
      The common user must have the permission to create the tool path and backup path.

    • package_path: path of the permission addition script add_sudo.sh. It must be an absolute path, for example, /home/username

      NOTE:
      This parameter must be set during manual installation. Otherwise, the profiling installation will fail.

    • use_eth0: whether to use eth0 for startup by default when ip is set to any

      NOTE:
      During the installation, the default value of use_eth0 is true. That is, when ip is set to any, eth0 is used for startup by default. If use_eth0 is set to false, you can select one from the IP addresses of multiple network adapters before startup.

    • load_date: whether to load data from the backup path. The default value is true, indicating that backup data is loaded from the backup path. If this parameter is set to false, the backup data is not loaded.

  3. Run the installation script.

    The scripts directory contains the mind_studio.sh file, which is the installation script. Perform 2 before running the script.

    • Run the following command to install Mind Studio and the DDK in a common scenario:

      bash mind_studio.sh install {absolute_ddk_path}

    • Run the following command to install Mind Studio separately when Mind Studio fails to be properly installed.

      bash mind_studio.sh install

    • Run the following command to install the DDK separately. If the DDK is not properly installed, you need to restart the Mind Studio after re-installing the DDK.

      bash mind_studio.sh installDDK {absolute_ddk_path}

    NOTE:
    {absolute_ddk_path} is the absolute path of the DDK package, for example, /home/username/ddk.tar.gz.

  4. Specify whether to back up the tools installation directory. （If the installation directory is not empty, this message is displayed.）

    [INFO] /home/username/tools is not empty. Files in the directory will be cleared during installation. Are you sure to back up them? [Y/N]: （Enter Y or y and press Enter to exit the installation. Back up the contents in /home/username/tools and then run the script again. Enter N or n to delete the contents in /home/username/tools and continue the installation.）
